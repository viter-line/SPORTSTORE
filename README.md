📂 backend/ — структура магазину
1.	requirements.txt (Список інструментів) - текстовий файл, який містить перелік усіх сторонніх бібліотек (Flask, SQLAlchemy тощо), які потрібно встановити для роботи проекту.
2.	database.py (Міст до даних) - створення об'єкта бази даних (db). Це своєрідний "перемикач" або "міст", через який НАШ код Python спілкується з базою даних MySQL.
3.	models.py (Фундамент або "креслення") - опис структури таблиць. Тут визначаємо, яку інформацію ми зберігаємо про Користувача (User), Товар (Product) чи Кошик (Cart).
Класи в цьому файлі діють як сині креслення (blueprints) для об'єктів. Ви один раз описуєте, що у товара є "назва" та "ціна", і система знає, як створити тисячі таких товарів.
4.	auth.py (Охорона та реєстрація) - усе, що стосується користувачів: реєстрація нових людей, перевірка паролів під час входу та видача "ключів" (JWT токенів) для доступу.
5.	products.py (Вітрина магазину) - логіка роботи з каталогом спортивних товарів. Обробляє запити на перегляд списку товарів або отримання деталей про конкретний м'яч чи кросівки.
6.	cart.py (Кошик та калькулятор) - управління покупками. Додавання товару в кошик, його видалення та, найголовніше, розрахунок загальної вартості (total price) перед покупкою.
7.	app.py (Головний менеджер) - "мозок" або "керівник проекту". Він запускає весь додаток, підключає налаштування та збирає всі інші файли (auth, products, cart) разом, щоб вони працювали як єдине ціле.
